# AS.D.003: Роль ≠ Режим вызова

## Различение

| | Роль | Режим вызова |
|-|------|-------------|
| **Что определяет** | *Что* проверяет, *какие* эталоны, *какой* формат выхода | *Как* запускается, *когда*, *кем* |
| **Меняется?** | Нет — фиксирована для задачи | Да — один и тот же Верификатор может работать в разных режимах |
| **Пример** | VR.R.001 Верификатор | Дневной sub-agent / Ночной headless CLI |

## Следствие: «Reviewer» — не отдельная роль

В DS-autonomous-agents/PROCESSES.md есть S51 «Night Review». Это **не отдельная роль** — это Верификатор (VR.R.001) в ночном режиме вызова.

| Термин | Что на самом деле |
|--------|------------------|
| «Reviewer» | Верификатор, вызванный ночью над коммитами дня |
| «Code verifier» | Верификатор, вызванный в сессии над текущим diff |
| «Capture reviewer» | Верификатор, вызванный над capture-candidates |
| «Приёмщик» | Верификатор при Close (VR.M.003) |

Одна роль → один промпт (с плейсхолдерами) → разные контексты и режимы.

## Два режима вызова

| Режим | Запуск | Контекст | Результат |
|-------|--------|----------|-----------|
| **Дневной (sub-agent)** | Agent tool внутри сессии Claude Code | Текущий diff + эталон. Context isolation от основной сессии | Verdict в чат → решение пользователя |
| **Ночной (headless)** | `claude -p "$prompt"` из bash-скрипта | Коммиты за день + эталон. Полная изоляция | Файл в `night-results/` → утренний ревью |

## Критерий проверки

> Если ты создаёшь нового «агента» — проверь: это действительно **новая роль** (другие ответственности, другой эталон) или **другой режим вызова** существующей роли? Если эталон и чеклист те же — это режим, не роль.
