# AS.D.002: Агент = Исполнитель + Роль

## Различение

| | Неправильно | Правильно |
|-|-------------|-----------|
| **Именование** | «Агент Scout», «агент Reviewer» | «Claude Code в роли Разведчика» (коротко: Разведчик) |
| **Сущность** | Отдельные программы/системы | Один исполнитель (Claude Code) + разные роли |
| **Аналогия** | Разные сотрудники | Один сотрудник в разных ролях (утром — аналитик, днём — разработчик) |

## Формула

```
Агент = Исполнитель (Claude Code) + Роль (из каталога DP.ROLE.001)
                                    + Промпт (контекст роли)
                                    + Режим (interactive / headless)
```

**Исполнитель** фиксирован: Claude Code (CLI или sub-agent).
**Роль** определяет: что проверять, какие эталоны, какой чеклист, какой формат выхода.
**Режим** определяет: дневной sub-agent в сессии / ночной headless через CLI.

## Следствия

1. **Именование по роли, не по функции:** «Разведчик», «Верификатор», «Стратег» — не «scout-agent», «reviewer-agent», «strategy-bot»
2. **Один промпт — один агент:** Каждый «агент» = промпт + agent-card. Не отдельная кодовая база
3. **Переиспользование исполнителя:** Один Claude Code может последовательно быть Разведчиком, потом Верификатором — меняется только загруженный контекст
4. **Режим ≠ роль:** Верификатор днём (sub-agent) и Верификатор ночью (headless) — одна роль, два режима вызова

## Связь с DP.D.033 (Роль ≠ Исполнитель)

DP.D.033 утверждает: роль — это набор ответственностей, а не конкретный человек/система. AS.D.002 — прямое следствие: «агент» — это не система, а роль, которую исполняет Claude Code.

## Критерий проверки

> Если ты говоришь «агент X», замени на «Claude Code в роли X» (или коротко «X»). Если предложение теряет смысл — ты смешиваешь агента с системой.
